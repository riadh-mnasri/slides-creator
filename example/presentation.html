Ce que fait l'outil

Fonctionnalités clés

  • Un fichier HTML par slide, triés alphabétiquement ou dans l'ordre de votre choix
  • Styles CSS scopés par slide — zéro conflit entre les pages
  • Feuilles de style externes inlinées automatiquement
  • Typographie premium : Inter + Playfair Display via Google Fonts
  • Transitions et animations configurables
  • Fichier de sortie autonome, aucune installation pour le visionner

Guide rapide

Comment utiliser

Depuis un dossier (trié alphabétiquement) :

node slides-creator.js --input ./mes-pages --output slides.html

Avec des fichiers explicites (ordre libre) :

node slides-creator.js intro.html methode.html conclusion.html

Toutes les options :

--title "Titre de la présentation"
--transition  fade | slide | zoom | convex | none
--slide-number           affiche le numéro de slide
--no-source-styles       ignore le CSS des pages source
--reveal-theme white     surcharge le thème Reveal.js

Philosophie

Pourquoi HTML comme source ?

HTML est le format de contenu le plus universel qui existe. Tout le monde sait l'écrire, tout éditeur le comprend, et tout navigateur le rend fidèlement.

Plutôt que d'imposer un format propriétaire, Slides Creator prend ce que vous avez déjà et le transforme en présentation professionnelle — sans friction.

  • Vos pages web existantes deviennent des slides
  • Votre CMS, votre documentation, vos exports HTML sont compatibles
  • Le résultat s'ouvre dans n'importe quel navigateur

Processus

Comment nous travaillons

  1. Analyse — Audit des pages HTML existantes et identification des contenus clés
  2. Structuration — Organisation logique des slides et hiérarchie de l'information
  3. Génération — Production du fichier de présentation en une commande
  4. Itération — Ajustements rapides grâce au rechargement instantané

Chaque étape prend moins de 5 minutes pour une présentation de 20 slides.

Aperçu technique

Architecture de l'outil

  • Parsing HTML — extraction du contenu avec Cheerio
    • Corps de la page (<body>)
    • Styles inline et fichiers CSS relatifs
  • Scoping CSS — isolation des styles par slide
    • Les règles body / html sont filtrées
    • Sélecteurs préfixés par [data-slide-id]
  • Rendu Reveal.js — présentation HTML autonome
    • Chargé via CDN, aucune dépendance locale pour visionner
    • Thème élégant injecté en priorité
